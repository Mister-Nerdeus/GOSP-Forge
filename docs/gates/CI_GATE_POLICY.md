# CI Gate Policy

CI complements local evidence and runs the repository truth gates. It may write non-secret workflow evidence under `artifacts/ci/` for upload as a GitHub Actions artifact.

## CI Gates

- Runtime policy verification: `node scripts/controls/verify-runtime-version.mjs`
- Frozen install: `pnpm install --frozen-lockfile`
- Static and package gates: `pnpm lint`, `pnpm -r build`, `pnpm -r typecheck`, `pnpm -r test`
- Dependency audit: `pnpm audit`
- Foundation proof commands: `pnpm validate:examples`, `pnpm simulate:clean-water`, `pnpm estimate:clean-water`, `pnpm run audit`
- CI evidence collection: `node scripts/controls/write-ci-evidence.mjs`

`pnpm audit` is pnpm's dependency-vulnerability audit. The repository foundation audit is a package script and must be invoked as `pnpm run audit` or via `pnpm --filter @gosp/cli start audit foundation`.

## Local Validation Separation

CI intentionally does not run `node scripts/controls/write-local-validation.mjs` because that command writes local currentness evidence under `artifacts/controls/local-validation/`. Local validation currentness is checked locally and may be attached to review evidence separately.

CI evidence is generated by `node scripts/controls/write-ci-evidence.mjs` under `artifacts/ci/`. The script records runtime proof, validation output, Clean Water simulation summary, estimate quality report, foundation audit output, git SHA, branch, command status, and non-claim text.

## Current Implementation Limits

This document describes foundation policy and contract intent. The current repository provides foundation packages, example fixtures, validation gates, and CI proof commands only. It does not claim production readiness, professional approval, potable-water certification, or production manufacturing approval.

## CI Artifacts

CI uploads `artifacts/ci/` and batch artifact markdown outputs as non-secret workflow evidence using `actions/upload-artifact`. These artifacts are found on the completed GitHub Actions run under the run's Artifacts section.

CI artifacts do not contain secrets, do not write to repository history, and do not create production, professional, potable-water, manufacturing, storage, hosting, marketplace, leaderboard, or release-approval claims.
