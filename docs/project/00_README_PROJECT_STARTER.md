# GOSP Forge Project Starter

**Project name:** GOSP Forge  
**Subtitle:** The open-source STEM systems simulator  
**Tagline:** Design it. Simulate it. Build it. Print it. Machine it. Score it. Remix it. Credit the creator.

## Purpose of this document set

This package defines the thesis, roadmap, architecture, contracts, invariants, AI constraints, Codex execution rules, governance, finance model, sponsorship model, and first implementation plan for **GOSP Forge**.

It is intended to be used as the starting material for a new ChatGPT Project, Codex project, GitHub repo roadmap, or technical steering document set.

## Core thesis

GOSP Forge is an open modular STEM problem-solving engine. It turns real-world problems into measurable, simulated solution spaces by combining:

- manufactured product specifications
- custom-designed parts
- software and logic modules
- physical materials
- manufacturing processes
- labor and schedule models
- physics and engineering approximations
- real-world pricing
- fabrication constraints
- simulation confidence
- creator attribution
- open-source remix lineage

The product is not only a community simulator, city builder, CAD tool, or game. It is a **problem-first, module-based, multi-fidelity simulation and pricing platform**.

## Immediate project state

The new `GOSP-Forge` repository currently exists as a clean starting point. It should be treated as the primary product repository.

The older `commons-sim` project should be treated as an incubator and source of reusable deterministic simulation and contract foundations. Selected pieces can be imported into GOSP Forge, but the new product thesis should not inherit the old narrow framing.

## Recommended first proof

The first complete proof should be:

# Automated Water Filter System

This example should include:

- manufactured products
- custom/fabricatable parts
- digital modules
- logic/control modules
- process modules
- resource flow
- materials
- BOM
- labor and manufacturing route
- cost estimate
- scorecard
- simulation confidence
- attribution
- safety profile
- education profile
- optional 3D/proxy representation

This proves the full loop:

```text
Problem → Modules → System → Simulation → Cost → Labor → Fabrication → Score → Credit → Remix
```

## Non-negotiable principle

> Public school teachers and students should be able to use the core educational platform for free. Commercial users, manufacturers, sponsors, grants, pro services, and optional paid infrastructure should fund the commons.

## Sponsor rule

> Sponsors can support the commons. They cannot own the commons.

## Simulation rule

> The simulator should provide the most realistic result possible for the available inputs, model fidelity, validation evidence, and declared assumptions. It must never overstate confidence.
